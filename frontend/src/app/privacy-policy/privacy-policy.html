2 What Information We Collect

2.1   Account & Authentication Information

When you register or sign in we collect and store:

  • Email address — used as your account identifier and to deliver price-drop notifications.
  • Display name / username — shown in the interface; sourced from your Keycloak profile.
  • User ID (UUID) — an opaque identifier generated by the authentication provider (Keycloak). Never linked to external tracking systems.
  • Authentication tokens — short-lived JWT access tokens and refresh tokens are stored locally in your browser (extension storage / session cookies). They are used exclusively to authenticate API requests to the Service and are never transmitted to third parties.
  • Session cookie — an encrypted, HTTP-only server-side session cookie issued by the auth gateway. It is bound to the price-tracker.gavaller.com domain and expires when you log out or close your browser session (configurable).
Matches Chrome Web Store declaration: Authentication information

2.2   Price Alert Data

When you create a price alert we store:

  • Tesco product ID (TPNC) — the numeric identifier of the product you wish to monitor.
  • Target price or percentage drop threshold — the condition you define for notification.
  • Alert status — enabled or disabled, last triggered timestamp.
  • Your user ID — to associate the alert with your account.

This data is stored in our MongoDB database on the Gavaller private server.

2.3   Email Notification Content

When a price alert is triggered, we send an email to the address registered on your account. The email contains the product name, old price, new price, and a link to the product. We do not store the content of outgoing emails beyond the delivery logs kept by our transactional email provider (Resend) for up to 30 days.

Matches Chrome Web Store declaration: Personal communications (emails)

2.4   Product Price Data (Scraped, Not Personal)

We collect publicly available product price data from the Tesco Hungary website (bevasarlas.tesco.hu) using an automated scraper. This data includes product names, prices, Clubcard prices, and identifiers. It is not personal data and is not linked to any individual user.

2.5   Browser Extension Local Storage

The browser extension stores the following data exclusively on your device using the browser's built-in storage.local API:

  • Language preference (EN or HU)
  • Theme preference (dark or light)
  • Product name cache — a map of product IDs to display names, capped at 200 entries, used to render the popup without extra network requests
  • Authentication tokens — access and refresh tokens as described in §2.1 above

None of this local data is transmitted to us or any third party, except that the access token is sent as a Bearer header to our own API endpoints to authenticate your requests.

9 Security Measures

  • All data in transit is encrypted using TLS 1.2+ (HTTPS enforced).
  • Authentication is handled via OAuth 2.0 / OpenID Connect (OIDC) with PKCE — passwords are never processed or stored by this application.
  • Session cookies are HTTP-only, Secure, and SameSite=Strict.
  • JWT access tokens are short-lived (typically 15 minutes); refresh tokens are rotated on use.
  • The backend infrastructure runs on a private server not exposed to shared cloud environments.
  • MongoDB is not publicly accessible; it is only reachable within the private Docker network.
